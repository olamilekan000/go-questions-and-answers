# Working with Different Data Types – Questions

## Question 1 – How does Go categorize its core data types?
Describe the main categories of data types in Go and give examples of each.

## Question 2 – What are the different ways to declare variables in Go?
Explain the `var` keyword with and without explicit types, and the short declaration operator `:=`, including when each form is appropriate.

## Question 3 – What are zero values and why do they matter?
What is a zero value in Go, and what are the typical zero values for basic types like numbers, strings, and booleans?

## Question 4 – How do variables differ from constants in Go?
Compare variables and constants in Go in terms of mutability, declaration, and where they can appear.

## Question 5 – How does Go handle unused variables and imports?
What happens if you declare a variable or import a package but never use it, and why does Go enforce this rule?

## Question 6 – How are strings represented and manipulated in Go?
Explain how strings are represented under the hood, how raw string literals differ from interpreted string literals, and how special characters like newlines are handled.

## Question 7 – How do you work with Unicode and runes in Go?
Describe how Go deals with Unicode strings, what a rune is, and how to correctly count characters rather than bytes.

## Question 8 – How is type conversion done in Go’s strong type system?
Explain how to discover the type of a value, and how to convert between types such as strings, integers, floats, and booleans using the standard library.

## Question 9 – How do you safely parse user input into numeric types?
Given user input as a string, how would you robustly convert it into an integer and handle invalid input?

## Question 10 – What is string interpolation and how do you do it in Go?
Describe how to build formatted strings that combine values of different types, and why functions like `fmt.Sprintf` are preferred over manual concatenation.
