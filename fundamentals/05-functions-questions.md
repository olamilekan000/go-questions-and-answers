# Grouping Code into Functions – Questions

## Question 1 – Why are functions important for structuring Go programs?
Explain how functions help with readability, reuse, and testing in Go applications.

## Question 2 – How do you declare a basic function in Go?
Show the general syntax for defining a function, including the function name, parameters, and return type.

## Question 3 – How are parameters and multiple parameters handled?
Describe how to declare functions with one or more parameters, and how parameter names and types are written.

## Question 4 – What is the difference between passing by value and by pointer?
Explain how arguments are passed to functions in Go and when you would choose to pass a pointer instead of a value.

## Question 5 – How do functions return values, including multiple values?
Describe how to return single and multiple values from a function, and why multiple return values are a common pattern in Go.

## Question 6 – What are named return values and when might they be used?
Explain what named return values are, how they change the way you write `return` statements, and any trade‑offs they introduce.

## Question 7 – How do variadic functions work in Go?
Describe what a variadic function is, how to declare one, and when it can be useful.

## Question 8 – What are anonymous functions and closures?
Explain how to define anonymous functions, how closures capture surrounding variables, and give an example use case.

## Question 9 – How can functions be used as values?
Discuss how functions can be assigned to variables, passed as arguments, and returned from other functions in Go.

## Question 10 – How does functional composition or “filtering” show up in Go?
Using Go’s functions and closures, outline how you might build a reusable `filter`‑style function that works on slices.
