# Structs and Data Blueprints – Questions

## Question 1 – What is a struct in Go and when would you use one?
Explain how structs model related data and compare them to maps or basic types.

## Question 2 – How do you declare and initialize structs?
Show how to define a struct type and create struct values using different initialization styles.

## Question 3 – How does copying a struct work?
Describe what happens when you assign one struct variable to another and how this differs from copying a pointer.

## Question 4 – How do you define methods on struct types?
Explain how to attach methods to a struct type using receivers, and when you would choose a value receiver versus a pointer receiver.

## Question 5 – How can structs be nested or composed?
Describe how one struct can contain another as a field and how this supports composition.

## Question 6 – How do you compare structs for equality?
Explain when two struct values can be compared with `==` and what constraints apply to the fields.

## Question 7 – What is the relationship between structs and JSON or database models?
Discuss why structs are often used as blueprints for JSON payloads or rows in a database.

## Question 8 – How do you update fields on a struct?
Show how to modify struct fields directly and through helper functions or methods.

## Question 9 – How can structs work with interfaces?
Explain how a struct type can satisfy an interface by implementing its methods, even without an explicit declaration.

## Question 10 – What design considerations apply when defining struct types?
Outline some best practices for naming, field visibility (exported vs. unexported), and keeping structs focused.
